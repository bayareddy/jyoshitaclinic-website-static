Act as a senior software architect and product designer.

I am building a scalable, production-ready Multi-Hospital SaaS application (tenant-based system). The system should support multiple hospitals (tenants) using a shared backend with strict data isolation.

Define detailed functionality, workflows, APIs, and database structure for the following modules:

----------------------------------------
1. Auth Module (Authentication & Authorization)
----------------------------------------
- User registration and login (JWT-based authentication)
- Role-based access control (RBAC): Super Admin, Hospital Admin, Doctor, Staff
- Multi-tenant login (user belongs to a specific hospital)
- Password reset, email/OTP verification
- Session management and token refresh
- Audit logs for login activity
- Permissions per role (fine-grained access)

----------------------------------------
2. Tenant (Hospital) Module
----------------------------------------
- Hospital registration (tenant onboarding)
- Unique tenant ID generation
- Hospital profile management (name, address, contact, departments)
- Subscription plans (Basic, Premium, Enterprise)
- Feature toggles per tenant
- Tenant-specific settings (working hours, billing rules)
- Data isolation strategy (shared DB + tenant_id)

----------------------------------------
3. Admin Module (Hospital Admin Control Panel)
----------------------------------------
- Admin dashboard:
  - Overview of hospital operations (patients, doctors, revenue, appointments)
  - Key KPIs and alerts

- Doctor onboarding:
  - Create doctor profiles
  - Assign specialization and department
  - Attach doctor to hospital (tenant)
  - Enable/disable doctor accounts

- Staff management:
  - Add/edit/remove staff (receptionist, nurse, lab technician, etc.)
  - Assign roles and responsibilities
  - Staff profile management

- Role & Permission Management (RBAC):
  - Create custom roles (e.g., Receptionist, Billing Executive, Lab Staff)
  - Assign permissions (module-level + action-level: create/read/update/delete)
  - Role hierarchy and inheritance (optional)
  - Map users to roles

- User management:
  - View all users (doctors + staff)
  - Activate/deactivate accounts
  - Reset passwords
  - Track user activity logs

- Department management:
  - Create/manage departments (Cardiology, Neurology, etc.)
  - Assign doctors to departments

- Scheduling oversight:
  - Manage doctor availability
  - Override schedules if needed

- Billing configuration:
  - Define service charges (consultation fees, procedures)
  - Configure tax (GST)
  - Set pricing rules

- Notifications & communication:
  - Send announcements to doctors/staff
  - System alerts and reminders

- Audit & compliance:
  - Track all admin actions (audit logs)
  - Ensure compliance with hospital policies

----------------------------------------
4. Doctor Module
----------------------------------------
- Doctor profile management:
  - Name, specialization, qualifications, experience
  - Contact details and profile photo
  - Department mapping
- Unique doctor ID per tenant
- Doctor onboarding (by admin)
- Role assignment (Doctor role in RBAC)
- Availability & scheduling
- Appointment management (view, accept, reschedule)
- Patient interaction (EMR access, prescriptions, notes)
- Telemedicine support (optional)
- Performance tracking
- Notifications

----------------------------------------
5. Patient Module
----------------------------------------
- Patient registration
- Unique patient ID per tenant
- Patient search and filtering
- EMR (medical records)
- Visit history
- Document uploads
- Patient status tracking
- Emergency contacts

----------------------------------------
6. Appointment Module
----------------------------------------
- Doctor scheduling integration
- Appointment booking (online/offline)
- Status tracking
- Queue management
- Calendar view
- Notifications
- Walk-in handling

----------------------------------------
7. Billing Module
----------------------------------------
- Invoice generation
- Service-based billing
- GST calculation
- Payment modes (cash/card/UPI)
- Payment tracking
- Insurance support (optional)
- Discounts/coupons
- Billing history

----------------------------------------
8. Reports Module
----------------------------------------
- Financial reports
- Patient analytics
- Appointment insights
- Doctor performance reports
- Export (PDF/Excel)
- Dashboard analytics
- Audit/system usage logs

----------------------------------------
Technical Requirements:
----------------------------------------
- Backend: NestJS (modular architecture)
- Frontend: Next.js (modern SaaS UI)
- Database: PostgreSQL (shared schema, tenant_id)
- Authentication: JWT + refresh tokens
- API Design: RESTful or GraphQL
- Deployment: Docker + CI/CD
- Security: Data isolation, encryption, validation
- Scalability: Support 10–20 hospitals initially, scalable to 100+

----------------------------------------
Deliverables:
----------------------------------------
1. Module-wise API endpoints (CRUD + advanced operations)
2. Database schema (tables, relationships, ER diagram description)
3. Folder structure (backend + frontend)
4. Key workflows (step-by-step)
5. Best practices for multi-tenancy and scalability
6. Optional: Microservices readiness design

Make the design practical, production-ready, and optimized for real hospital workflows.
